## 1.1 LatentMAS 是什么？

**LatentMAS**（Latent Multi-Agent System）是一个多智能体推理框架，其核心创新在于将 Agent 之间的协作从 **Token 空间** 转移到 **模型潜在空间（Latent Space）**。

### 传统方法 vs LatentMAS

| 对比维度 | 传统方法（TextMAS） | LatentMAS |
|---------|-------------------|-----------|
| Agent 通信方式 | 通过文本传递信息 | 通过潜在向量传递信息 |
| Token 消耗 | 较多 | 减少 50-80% |
| 推理速度 | 较慢 | 快 3-7 倍 |
| 内存效率 | 一般 | 更高（复用 KV Cache）|

### 核心特性

1. **高效（Efficient）**：大幅减少多步推理的 Token 使用量
2. **无需训练（Training-free）**：潜在空间对齐机制确保稳定生成
3. **通用（General）**：兼容任意 HuggingFace 模型，支持 vLLM 加速

## 1.3 环境配置

### 硬件要求

- **GPU**：建议 NVIDIA GPU，显存 >= 16GB（运行 Qwen3-14B）
- **内存**：>= 16GB RAM
- **硬盘**：>= 50GB 可用空间（用于模型缓存）

### 软件要求

- **Python**：3.8 - 3.10（推荐 3.10）
- **CUDA**：11.8 或更高版本
- **操作系统**：Linux/macOS/Windows（WSL2 最佳）

### 快速安装步骤

#### 步骤 1：创建 conda 环境

```bash
conda create -n latentmas python=3.10 -y
conda activate latentmas
```

#### 步骤 2：安装基础依赖

```bash
pip install -r requirements.txt
```

`requirements.txt` 内容：

```
transformers  # HuggingFace 模型库
torch         # PyTorch 深度学习框架
numpy         # 数值计算
tqdm          # 进度条
accelerate    # 分布式/加速训练
datasets      # HuggingFace 数据集库
```

#### 步骤 3：（可选）安装 vLLM 加速

如果你需要更快的推理速度（推荐）：

```bash
pip install vllm
```

> 注意：vLLM 需要较新的 NVIDIA 驱动和 CUDA 版本。

#### 步骤 4：配置 HuggingFace 缓存目录（推荐）

设置环境变量避免重复下载模型：

```bash
# Linux/macOS
export HF_HOME=/path/to/huggingface
export TRANSFORMERS_CACHE=$HF_HOME
export HF_DATASETS_CACHE=$HF_HOME

# Windows (PowerShell)
$env:HF_HOME="D:\huggingface"
$env:TRANSFORMERS_CACHE=$env:HF_HOME
$env:HF_DATASETS_CACHE=$env:HF_HOME
```

#### 步骤 5：验证安装

```python
python -c "import transformers; import torch; import vllm; print('All packages imported successfully!')"
```

## 1.4 快速运行示例

安装完成后，让我们运行一个最简单的例子。

### 运行基线方法（单 Agent）

```bash
python run.py \
    --method baseline \
    --model_name Qwen/Qwen3-4B \
    --task gsm8k \
    --max_samples 5 \
    --max_new_tokens 1024
```

参数说明：

| 参数 | 含义 | 示例值 |
|------|------|--------|
| `--method` | 推理方法 | `baseline` / `text_mas` / `latent_mas` |
| `--model_name` | 模型名称 | `Qwen/Qwen3-4B` / `Qwen/Qwen3-14B` |
| `--task` | 任务/数据集 | `gsm8k` / `aime2024` / `arc_easy` |
| `--max_samples` | 最大样本数 | `-1` 表示全部 |
| `--max_new_tokens` | 最大生成 Token 数 | `1024` / `2048` |

### 运行 LatentMAS 方法

```bash
python run.py \
    --method latent_mas \
    --model_name Qwen/Qwen3-4B \
    --task gsm8k \
    --prompt sequential \
    --max_samples 5 \
    --max_new_tokens 1024
```

额外参数说明：

| 参数 | 含义 |
|------|------|
| `--prompt` | Agent 架构：`sequential`（顺序）或 `hierarchical`（层级）|
| `--latent_steps` | 潜在推理步数（默认 0-80 之间调优）|
| `--use_vllm` | 使用 vLLM 加速 |

## 1.8 常见问题

### Q1：没有 GPU 能运行吗？

可以运行，但速度会很慢。建议使用 GPU，或者使用 Google Colab（提供免费 GPU）。

### Q2：模型下载太慢怎么办？

设置国内镜像源：

```bash
pip install -i https://mirrors.aliyun.com/pypi/simple/ -r requirements.txt
```

### Q3：vLLM 安装失败怎么解决？

vLLM 对 CUDA 版本有要求。建议使用官方推荐的 CUDA 版本，或查看 [vLLM 官方安装指南](https://docs.vllm.ai/en/latest/getting_started/installation.html)。

### Q4：显存不足怎么办？

- 减小 `max_new_tokens` 参数
- 使用更小的模型（如 `Qwen/Qwen3-4B` 代替 `Qwen/Qwen3-14B`）
- 减小 `batch_size`（当前代码自动处理）
