### 4.2.1 set_seed() —— 设置随机种子

```python
def set_seed(seed: int) -> None:
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)
    os.environ["PYTHONHASHSEED"] = str(seed)
```

**为什么需要设置随机种子？**

深度学习中的随机性来源：
1. **Python 内置随机**：`random.random()`
2. **NumPy 随机**：`np.random.randn()`
3. **PyTorch 随机**：`torch.randn()`
4. **CUDA 随机**：GPU 上的随机操作

**设置种子后**：相同的输入 + 相同的种子 = 相同的结果

**重要性**：确保实验可复现，方便对比不同方法的效果。

### 4.2.2 auto_device() —— 自动选择设备

```python
def auto_device(device: Optional[str] = None) -> torch.device:
    if device is not None:
        return torch.device(device)      # 优先使用指定的设备
    if torch.cuda.is_available():
        return torch.device("cuda")      # 其次使用 GPU
    return torch.device("cpu")          # 最后使用 CPU
```

**返回值类型**：`torch.device`（PyTorch 的设备对象）

**使用示例**：

```python
# 手动指定设备
device = auto_device("cuda:0")

# 自动选择
device = auto_device()  # 如果有 GPU 返回 cuda，否则返回 cpu
```

**设备类型**：
| 设备 | 格式 | 说明 |
|------|------|------|
| CPU | `torch.device("cpu")` | 中央处理器 |
| GPU 0 | `torch.device("cuda:0")` | 第一块 NVIDIA 显卡 |
| GPU 1 | `torch.device("cuda:1")` | 第二块 NVIDIA 显卡 |

## 4.4 分词器相关

### 4.4.1 什么是 Pad Token？

分词器（Tokenizer）处理批量数据时，需要将不同长度的序列**填充（Pad）**到相同长度。

**问题**：填充的 token 应该是什么？

**解决方案**：使用 `pad_token`
- 如果 tokenizer 有 `pad_token_id`，直接使用
- 如果没有，尝试使用 `eos_token`（结束 token）
- 如果都没有，添加一个特殊的 `<pad>` token

```python
def _ensure_pad_token(tokenizer: AutoTokenizer) -> None:
    if tokenizer.pad_token_id is None:
        if tokenizer.eos_token is not None:
            tokenizer.pad_token = tokenizer.eos_token
        else:
            tokenizer.add_special_tokens({"pad_token": "<pad>"})
```

### 4.4.2 Chat Template

现代语言模型使用特殊的 **Chat Template** 来格式化对话：

```python
# 原始消息格式
messages = [
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么可以帮助你的吗？"}
]

# 使用 chat_template 渲染后
prompt_text = tokenizer.apply_chat_template(messages, tokenize=False)
# 输出：
# <|user|>\n你好\n</|user|>\n<|assistant|>\n
```

**render_chat() 方法**：

```python
def render_chat(self, messages: List[Dict], add_generation_prompt: bool = True) -> str:
    tpl = getattr(self.tokenizer, "chat_template", None)
    if tpl:
        return self.tokenizer.apply_chat_template(
            messages, tokenize=False, add_generation_prompt=add_generation_prompt
        )
    # Fallback: 手动拼接
    segments = []
    for message in messages:
        role = message.get("role", "user")
        content = message.get("content", "")
        segments.append(f"<|{role}|>\n{content}\n</|{role}|>")
    if add_generation_prompt:
        segments.append("<|assistant|>")
    return "\n".join(segments)
```

## 4.6 核心方法详解

### 4.6.1 prepare_chat_input() —— 准备单条输入

**输入**：聊天消息列表
**输出**：`(prompt_text, input_ids, attention_mask, tokens)`

```python
def prepare_chat_input(
    self,
    messages: List[Dict],
    add_generation_prompt: bool = True
) -> Tuple[str, torch.Tensor, torch.Tensor, List[str]]:
    # 1. 渲染为文本
    prompt_text = self.render_chat(messages, add_generation_prompt=add_generation_prompt)

    # 2. 分词为张量
    encoded = self.tokenizer(
        prompt_text,
        return_tensors="pt",     # 返回 PyTorch 张量
        add_special_tokens=False,
    )
    input_ids = encoded["input_ids"].to(self.device)
    attention_mask = encoded["attention_mask"].to(self.device)

    # 3. 获取 token 列表（用于调试）
    active_ids = input_ids[0][attention_mask[0].bool()].tolist()
    tokens = self.tokenizer.convert_ids_to_tokens(active_ids)

    return prompt_text, input_ids, attention_mask, tokens
```

**返回值说明**：
| 返回值 | 类型 | 说明 |
|--------|------|------|
| `prompt_text` | str | 渲染后的提示文本 |
| `input_ids` | Tensor [1, seq_len] | 输入 token ID |
| `attention_mask` | Tensor [1, seq_len] | 注意力掩码 |
| `tokens` | List[str] | token 字符串列表 |

### 4.6.2 prepare_chat_batch() —— 准备批量输入

**输入**：批量聊天消息列表
**输出**：`(prompts, input_ids, attention_mask, tokens_batch)`

```python
def prepare_chat_batch(
    self,
    batch_messages: List[List[Dict]],
    add_generation_prompt: bool = True,
) -> Tuple[List[str], torch.Tensor, torch.Tensor, List[List[str]]]:
    prompts: List[str] = []
    for messages in batch_messages:
        prompts.append(self.render_chat(messages, add_generation_prompt=add_generation_prompt))

    # 批量分词（自动 padding）
    encoded = self.tokenizer(
        prompts,
        return_tensors="pt",
        padding=True,           # 自动填充到相同长度
        add_special_tokens=False,
    )
    input_ids = encoded["input_ids"].to(self.device)
    attention_mask = encoded["attention_mask"].to(self.device)

    # 转换为 token 列表
    tokens_batch: List[List[str]] = []
    for ids_row, mask_row in zip(input_ids, attention_mask):
        active_ids = ids_row[mask_row.bool()].tolist()
        tokens_batch.append(self.tokenizer.convert_ids_to_tokens(active_ids))

    return prompts, input_ids, attention_mask, tokens_batch
```

**与单条的区别**：
- 批量版本使用 `padding=True`
- 返回 `List[str]` 和 `List[List[str]]`

### 4.6.3 generate_text_batch() —— 文本生成

**输入**：输入 token ID、生成参数
**输出**：`(generations, past_key_values)`

```python
@torch.no_grad()
def generate_text_batch(
    self,
    input_ids: torch.Tensor,
    attention_mask: Optional[torch.Tensor] = None,
    *,
    max_new_tokens: int = 256,
    temperature: float = 0.7,
    top_p: float = 0.95,
    past_key_values: Optional[Tuple] = None,
) -> Tuple[List[str], Optional[Tuple]]:
```

**核心参数**：
| 参数 | 说明 |
|------|------|
| `input_ids` | 输入 token ID [batch, seq_len] |
| `max_new_tokens` | 最大生成 token 数 |
| `temperature` | 采样温度（控制随机性）|
| `top_p` | Nucleus 采样阈值 |
| `past_key_values` | KV Cache（用于增量生成）|

**生成过程**：

```python
# 1. 处理 attention_mask
if attention_mask is None:
    attention_mask = torch.ones_like(input_ids, device=self.device)
prompt_lengths = attention_mask.sum(dim=1).tolist()

# 2. 处理 KV Cache（增量生成）
cache_position = None
if past_key_values is not None:
    past_len = _past_length(past_key_values)
    cache_position = torch.arange(past_len, past_len + input_ids.shape[-1], ...)
    # 合并历史 mask 和当前 mask
    attention_mask = torch.cat([past_mask, attention_mask], dim=-1)

# 3. 调用模型生成
outputs = self.model.generate(
    input_ids=input_ids,
    attention_mask=attention_mask,
    max_new_tokens=max_new_tokens,
    temperature=temperature,
    top_p=top_p,
    do_sample=True,
    pad_token_id=self.tokenizer.pad_token_id,
    return_dict_in_generate=True,
    output_scores=False,
    past_key_values=past_key_values,
    cache_position=cache_position,
)

# 4. 解码输出
sequences = outputs.sequences
generations: List[str] = []
for idx, length in enumerate(prompt_lengths):
    generated_ids = sequences[idx, length:]  # 去除输入部分
    text = self.tokenizer.decode(generated_ids, skip_special_tokens=True).strip()
    generations.append(text)

return generations, outputs.past_key_values
```

**KV Cache 机制**：

```
第一次生成：
Input: [A, B, C] → Output: [D, E]
Cache: [A, B, C, D, E]

第二次生成（复用 Cache）：
Input: [F, G] + Cache → Output: [H, I]
Cache: [A, B, C, D, E, F, G, H, I]
```

## 4.7 潜在空间重对齐（Latent Space Realignment）

### 4.7.1 为什么需要重对齐？

Transformer 的结构：
```
输入嵌入层 (Input Embedding) → Transformer 层 → 输出头 (Output Head)
     [vocab_size, hidden_dim]                      [hidden_dim, vocab_size]
```

**问题**：
- 输入嵌入和输出嵌入是**不同的权重矩阵**
- 在潜在空间生成的向量需要投影回输入空间
- 如果不做对齐，潜在向量可能"方向不对"

### 4.7.2 解决方案：Gram 矩阵求逆

```python
def _build_latent_realign_matrix(self, model, device, args) -> Tuple[torch.Tensor, torch.Tensor]:
    # 获取输入和输出嵌入权重
    input_weight = input_embeds.weight.detach()  # [vocab, hidden]
    output_weight = output_embeds.weight.detach() # [vocab, hidden]

    # 计算 Gram 矩阵并正则化
    gram = torch.matmul(output_weight.T, output_weight)  # [hidden, hidden]
    gram = gram + 1e-5 * torch.eye(gram.shape[0])        # 正则化

    # 求解线性系统
    rhs = torch.matmul(output_weight.T, input_weight)    # [hidden, vocab]
    realign_matrix = torch.linalg.solve(gram, rhs)       # [hidden, vocab]

    # 计算目标范数
    target_norm = input_weight.norm(dim=1).mean()

    return realign_matrix, target_norm
```

### 4.7.3 应用重对齐

```python
def _apply_latent_realignment(self, hidden: torch.Tensor, model: torch.nn.Module) -> torch.Tensor:
    matrix, target_norm = self._ensure_latent_realign_matrix(model, hidden.device, self.args)

    # 投影到输入空间
    hidden_fp32 = hidden.to(torch.float32)
    aligned = torch.matmul(hidden_fp32, matrix)  # [batch, vocab]

    # 范数对齐
    aligned_norm = aligned.norm(dim=-1, keepdim=True).clamp_min(1e-6)
    aligned = aligned * (target_norm / aligned_norm)

    return aligned.to(hidden.dtype)
```

## 4.9 常见问题

### Q1：CUDA out of memory 怎么办？

**解决方案**：
1. 减小 `batch_size`
2. 使用更小的模型（如 `Qwen3-4B` 代替 `Qwen3-14B`）
3. 使用 vLLM 后端（显存利用率更高）
4. 减小 `max_new_tokens`

### Q2：vLLM 安装失败？

**原因**：vLLM 对 CUDA 版本有严格要求。

**解决方案**：
1. 确认 CUDA 版本：`nvcc --version`
2. 安装匹配的 vLLM 版本
3. 或使用 transformers 后端（更通用）

### Q3：模型下载很慢？

**解决方案**：
```bash
export HF_HOME=/path/to/huggingface_cache
# 或设置国内镜像
export HF_ENDPOINT=https://hf-mirror.com
```
