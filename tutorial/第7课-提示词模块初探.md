## 7.3 Agent 角色定义

### 7.3.1 四个核心 Agent

项目定义了 **4 种 Agent 角色**：

| Agent | 名称 | 职责 | 核心能力 |
|-------|------|------|----------|
| **Planner** | 规划者 | 设计解题步骤 | 分析问题、制定计划 |
| **Critic** | 批评者 | 评估计划质量 | 找出漏洞、提供反馈 |
| **Refiner** | 精炼者 | 改进优化计划 | 整合反馈、生成更好方案 |
| **Judger** | 判断者 | 给出最终答案 | 综合信息、输出结果 |

### 7.3.2 Agent 执行流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│Planner │────►│ Critic  │────►│ Refiner │────►│ Judger  │
│ 规划者  │     │ 批评者  │     │ 精炼者  │     │ 判断者  │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
     │               │                │               │
  设计计划        评估计划         改进计划        最终答案
     │               │                │               │
     ▼               ▼                ▼               ▼
  "第一步...    "计划合理，      "根据反馈，      "答案是..."
  第二步..."     但第二步...      改进如下..."
```

## 7.5 各 Agent 提示词详解

### 7.5.1 Planner（规划者）

**提示词（sequential_latent_mas）**：

```
You are a Planner Agent. Given an input question, design a clear,
step-by-step plan for how to solve the question.

Question: {question}

Your outlined plan should be concise with a few bulletpoints for each step.
Do not produce the final answer.

Now output your plan to solve the question below:
```

**职责**：
1. 理解问题
2. 设计解决步骤
3. 以简洁的要点形式输出
4. **不输出最终答案**

**输出格式**（期望）：
```
Planner Agent's Output:
- Step 1: ...
- Step 2: ...
- Step 3: ...
```

---

### 7.5.2 Critic（批评者）

**提示词（sequential_latent_mas）**：

```
Question: {question}

You are a Critic Agent to evaluate the correctness of the input plan
for the given question and provide helpful feedback for improving the plan.
The plan information is provided in latent KV representation format.
Review the plan and question and output:
(1) original plan contents
(2) constructive feedback on the original plan.

Format your response as follows:
Original Plan: [Copy the provided Planner Agent's plan here]
Feedback: [Your detailed feedback to improve the plan here]

Now, output your response below:
```

**职责**：
1. 评估计划的正确性
2. 指出潜在问题
3. 提供建设性反馈

**输出格式**（期望）：
```
Original Plan: [复述 Planner 的计划]
Feedback: [详细反馈，指出问题和建议]
```

---

### 7.5.3 Refiner（精炼者）

**提示词（sequential_latent_mas）**：

```
Question: {question}

You are a Refiner Agent to provide a refined step-by-step plan for
solving the given question.
You are provided with:
(1) latent-format information: a previous plan with feedback
(2) text-format information: the input question you need to solve.

Based on the input, write a refined and improved plan to solve the question.
Make sure your output plan is correct and concise.

Now, output your refined plan below:
```

**职责**：
1. 接收 Planner 的计划和 Critic 的反馈
2. 整合信息
3. 生成改进后的计划

**关键差异**：Refiner 能够看到反馈，而 Planner 不能。

### 7.5.4 Judger（判断者）

**提示词因任务而异**：

#### 7.5.4.1 数学任务（GSM8K, AIME）

```
Target Question: {question}

You are a helpful assistant. You are provided with latent information
for reference and a target question to solve.

The latent information might contain irrelevant contents.
Ignore it if it is not helpful for solving the target question.

You must reason step-by-step to solve the provided Target Question
without outputting other irrelevant information.

Now, reason step by step and output the final answer inside \boxed{YOUR_FINAL_ANSWER}.
```

#### 7.5.4.2 选择题任务（ARC, GPQA, MedQA）

```
Target Question: {question}

You must reason step-by-step to solve the provided Target Question
without outputting other irrelevant information.
Your final answer must be selected from A,B,C,D.
For example \boxed{A}. Do not add any other contents inside the box.

Now, reason step by step and output the final answer inside \boxed{YOUR_FINAL_ANSWER}.
```

#### 7.5.4.3 代码任务（MBPP+, HumanEval+）

```
Target Question: {question}

You must reason step-by-step to solve the provided Target Question
without outputting other irrelevant information.
You must put all python code as self-contained Python function in
markdown code blocks. For example:

```python
import math
def add(a, b):
    return a + b
```

Now, reason step by step and output the final answer inside

```python
YOUR_PYTHON_CODE
```.
```

## 7.6 Sequential vs Hierarchical 架构

### 7.6.1 Sequential（顺序架构）

**特点**：Agent 串行执行，上下文逐步累积

**流程**：
```
Planner → Critic → Refiner → Judger
   │          │         │         │
   │计划       │评估      │改进      │答案
   └──────────┴─────────┴─────────┘
                   │
              上下文累积
```

**提示词函数**：
- `build_agent_message_sequential_latent_mas()`
- `build_agent_messages_sequential_text_mas()`

### 7.6.2 Hierarchical（层级架构）

**特点**：不同 Agent 扮演不同"专家角色"，更高效

**流程**：
```
Planner (Math Agent) ─┐
Critic (Science Agent) ├─► Judger (Summarizer)
Refiner (Code Agent) ─┘
        │
   各自独立
   协同决策
```

**提示词函数**：
- `build_agent_message_hierarchical_latent_mas()`
- `build_agent_messages_hierarchical_text_mas()`

## 7.8 LatentMAS vs TextMAS 提示词差异

### 7.8.1 Critic Agent 对比

**TextMAS（文本传递）**：

```
You are a Critic Agent. You are provided with:
(1) the original question, and
(2) the Planner Agent's plan in text format.
```

**LatentMAS（潜在空间传递）**：

```
You are a Critic Agent to evaluate the correctness of the input plan
for the given question and provide helpful feedback for improving the plan.
The plan information is provided in latent KV representation format.
```

### 7.8.2 关键差异

| 对比项 | TextMAS | LatentMAS |
|--------|---------|-----------|
| 信息传递 | 文本 | 潜在向量 |
| Critic 描述 | "in text format" | "in latent KV representation format" |
| 信息处理 | 完整文本 | 压缩的潜在表示 |

**本质区别**：LatentMAS 告诉 Critic 信息来自 KV Cache，而非显式文本。

## 7.11 常见问题

### Q1：为什么需要 4 种 Agent 角色？

**原因**：模拟人类解决问题的协作过程。
- 一个人可能设计出有缺陷的计划
- 需要他人审视和改进
- 最终由经验丰富的人做决定

### Q2：能不能减少 Agent 数量？

**可以**，但效果会下降：
- 3 Agent：去掉 Critic → 反馈减少
- 2 Agent：去掉 Refiner → 改进减少
- 1 Agent：退化为 baseline

### Q3：提示词能改吗？

**可以**，但需要注意：
1. 保持角色定义清晰
2. 保持输出格式一致
3. 避免引入歧义

| Judger | 所有历史 + 问题 | 最终答案 |
