## 2.2 核心文件详解

### 2.2.1 run.py —— 主程序入口

`run.py` 是整个项目的入口文件，负责：
1. 解析命令行参数
2. 加载模型和数据
3. 选择并运行对应的方法
4. 输出评估结果

#### 核心函数

| 函数 | 行号 | 作用 |
|------|------|------|
| `evaluate()` | 26 | 计算准确率和正确数 |
| `process_batch()` | 33 | 处理一个批次的样本 |
| `main()` | 84 | 主函数，串联整个流程 |

#### 参数解析（argparse）

`run.py` 使用 Python 标准库 `argparse` 解析命令行参数：

```python
parser = argparse.ArgumentParser()

# 核心参数
parser.add_argument("--method", choices=["baseline", "text_mas", "latent_mas"], required=True)
parser.add_argument("--model_name", type=str, required=True)
parser.add_argument("--max_samples", type=int, default=-1)
parser.add_argument("--task", type=str, default="gsm8k")
parser.add_argument("--prompt", choices=["sequential", "hierarchical"], default="sequential")

# 生成参数
parser.add_argument("--max_new_tokens", type=int, default=4096)
parser.add_argument("--temperature", type=float, default=0.6)
parser.add_argument("--top_p", type=float, default=0.95)

# vLLM 参数
parser.add_argument("--use_vllm", action="store_true")
parser.add_argument("--device2", type=str, default="cuda:1")
```

#### 主流程（main 函数）

```
main() 执行流程：

1. 解析参数（argparse）
2. 设置随机种子（set_seed）
3. 选择设备（auto_device）
4. 初始化模型（ModelWrapper）
5. 选择方法（BaselineMethod / TextMASMethod / LatentMASMethod）
6. 加载数据集（根据 --task 选择加载函数）
7. 批量处理（process_batch 循环）
8. 输出结果（JSON 格式）
```

#### 输出格式

运行结束后，会输出 JSON 格式的结果：

```json
{
  "method": "latent_mas",
  "model": "Qwen/Qwen3-14B",
  "split": "test",
  "seed": 42,
  "max_samples": 100,
  "accuracy": 0.85,
  "correct": 85,
  "total_time_sec": 1234.5678,
  "time_per_sample_sec": 12.3457
}
```

**注**：如果你想在训练时使用 AIME 数据集，代码会自动加载 `train` 集（而非 `test` 集），因为 AIME 只有训练集。

### 2.2.3 prompts.py —— 提示词模板

`prompts.py` 定义了 Agent 的提示词模板，是多智能体系统的"灵魂"。

#### 四个核心 Agent 角色

| Agent | 角色 | 职责 |
|-------|------|------|
| **Planner** | 规划者 | 设计解题步骤和计划 |
| **Critic** | 批评者 | 评估计划并指出问题 |
| **Refiner** | 精炼者 | 根据反馈改进计划 |
| **Judger** | 判断者 | 最终给出答案 |

#### 两种架构

| 架构 | 特点 | 适用场景 |
|------|------|----------|
| **sequential**（顺序）| Agent 串行执行，上下文累积 | 简单任务、顺序推理 |
| **hierarchical**（层级）| 分层组织，效率更高 | 复杂任务、层级推理 |

### 2.2.4 data.py —— 数据加载

#### 支持的数据集


## 2.3 methods/ 目录详解

`methods/` 目录包含三种推理方法的实现，是项目的核心。

### 2.3.1 目录结构

```
methods/
├── __init__.py          # 定义 Agent 数据类
├── baseline.py          # 单 Agent 基线方法
├── text_mas.py          # 文本多 Agent 方法
└── latent_mas.py        # 潜在空间多 Agent 方法
```

### 2.3.2 Agent 数据类（methods/__init__.py）

```python
from dataclasses import dataclass
from typing import List

@dataclass
class Agent:
    name: str   # Agent 名称：Planner/Critic/Refiner/Judger
    role: str   # Agent 角色：planner/critic/refiner/judger

def default_agents() -> List[Agent]:
    return [
        Agent(name="Planner", role="planner"),
        Agent(name="Critic", role="critic"),
        Agent(name="Refiner", role="refiner"),
        Agent(name="Judger", role="judger"),
    ]
```

### 2.3.3 三种方法对比

| 方法 | 文件 | 通信方式 | Agent 协作 |
|------|------|----------|------------|
| **baseline** | `baseline.py` | 无 | 单 Agent 直接生成 |
| **text_mas** | `text_mas.py` | 文本传递 | 多 Agent 串行协作 |
| **latent_mas** | `latent_mas.py` | 潜在向量传递 | 多 Agent 潜在空间协作 |

#### baseline.py

最简单的基线方法，单 Agent 直接生成答案：

```
问题 → Model → 答案
```

#### text_mas.py

多 Agent 通过文本传递信息：

```
问题 → Planner → Critic → Refiner → Judger → 答案
         ↓         ↓         ↓         ↓
       文本       文本       文本       文本
```

#### latent_mas.py

多 Agent 通过潜在空间传递隐向量（核心创新）：

```
问题 → Planner → Critic → Refiner → Judger → 答案
         ↓         ↓         ↓         ↓
      潜在向量  潜在向量  潜在向量  潜在向量
```

## 2.9 本课小结

### 核心要点

| 要点 | 内容 |
|------|------|
| **run.py** | 主入口，负责参数解析、流程控制、结果输出 |
| **models.py** | 模型封装，支持 transformers 和 vLLM 两种后端 |
| **prompts.py** | Agent 提示词模板，定义 4 种角色和 2 种架构 |
| **data.py** | 数据加载，支持 7 种数据集 |
| **utils.py** | 工具函数：答案提取、标准化、超时执行 |
| **methods/** | 三种推理方法的实现目录 |

### 关键区别

- **baseline**：单 Agent，直接生成
- **text_mas**：多 Agent，文本传递
- **latent_mas**：多 Agent，潜在向量传递（核心创新）
