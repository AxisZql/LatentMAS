## 5.2 文本生成参数配置

### 5.2.1 核心生成参数

```python
def generate_text_batch(
    self,
    input_ids: torch.Tensor,
    attention_mask: Optional[torch.Tensor] = None,
    *,
    max_new_tokens: int = 256,      # 最大生成 token 数
    temperature: float = 0.7,        # 采样温度
    top_p: float = 0.95,             # Nucleus 采样
    past_key_values: Optional[Tuple] = None,  # KV Cache
) -> Tuple[List[str], Optional[Tuple]]:
```

### 5.2.2 max_new_tokens —— 最大生成长度

**作用**：限制生成的最大 token 数量

**示例**：
```python
# 生成最多 1024 个 token
generate_text_batch(..., max_new_tokens=1024)

# 生成最多 2048 个 token
generate_text_batch(..., max_new_tokens=2048)
```

**注意**：实际生成的 token 数可能小于这个值（如果遇到结束 token）

### 5.2.3 temperature —— 采样温度

**作用**：控制输出的**随机性**

| temperature 值 | 效果 | 适用场景 |
|----------------|------|----------|
| 0.0 | 贪婪解码（始终选最高概率）| 确定性问题 |
| 0.3 - 0.7 | 适度随机 | 通用生成 |
| 0.8 - 1.0 | 高度随机 | 创意生成 |
| > 1.0 | 极高随机 | 探索性任务 |

**数学原理**：

温度采样将 logits 除以温度值后再计算概率：

```python
probs = softmax(logits / temperature)
```

- `temperature → 0`：分布变"尖"，趋向确定
- `temperature → ∞`：分布变"平"，趋向均匀

### 5.2.4 top_p —— Nucleus 采样

**作用**：动态选择概率阈值为 p 的最小 token 集合

**示例**：
```python
# top_p=0.95：只从累计概率 > 95% 的 token 中采样
generate_text_batch(..., top_p=0.95)

# top_p=1.0：使用所有 token（等价于 temperature 采样）
generate_text_batch(..., top_p=1.0)
```

**图示**：

```
Token 概率分布（排序后）：
Token A: 0.50  ████████████
Token B: 0.25  ██████
Token C: 0.15  ████
Token D: 0.05  █
Token E: 0.05  █

top_p=0.90：
累计概率：A(0.50) + B(0.25) + C(0.15) = 0.90
→ 只从 A, B, C 中采样

top_p=0.95：
累计概率：A(0.50) + B(0.25) + C(0.15) + D(0.05) = 0.95
→ 从 A, B, C, D 中采样
```

### 5.2.5 temperature + top_p 组合

项目默认配置：
```python
temperature=0.6
top_p=0.95
```

**推荐组合**：
| 场景 | temperature | top_p |
|------|-------------|-------|
| 数学推理 | 0.3 - 0.5 | 0.9 - 0.95 |
| 代码生成 | 0.3 - 0.5 | 0.9 - 0.95 |
| 创意写作 | 0.7 - 0.9 | 0.95 - 1.0 |
| 问答 | 0.3 - 0.6 | 0.9 - 0.95 |

## 5.3 KV Cache 处理

### 5.3.1 什么是 KV Cache？

**KV Cache** 是一种加速生成的技术，缓存已计算的 Key-Value 注意力矩阵。

**无 Cache 的生成**：
```
Step 1: 输入 [A, B] → 计算 K, V → 输出 D
Step 2: 输入 [A, B, C] → 重新计算 K, V (A, B, C) → 输出 E  ❌ 重复计算
Step 3: 输入 [A, B, C, D] → 重新计算 K, V (A, B, C, D) → 输出 F  ❌ 重复计算
```

**有 Cache 的生成**：
```
Step 1: 输入 [A, B] → 计算 K, V → 输出 D → Cache = [A, B, D]
Step 2: 输入 [C] + Cache → 只需计算 C 的 K, V → 输出 E → Cache = [A, B, D, E]  ✅
Step 3: 输入 [F] + Cache → 只需计算 F 的 K, V → 输出 G → Cache = [A, B, D, E, F, G]  ✅
```

### 5.3.2 代码中的 KV Cache 处理

```python
def _past_length(past_key_values: Optional[Tuple]) -> int:
    if not past_key_values:
        return 0
    k = past_key_values[0][0]
    return k.shape[-2]
```

**增量生成时合并 mask**：

```python
if past_key_values is not None:
    past_len = _past_length(past_key_values)

    # 历史 mask（全 1）
    past_mask = torch.ones(
        (attention_mask.shape[0], past_len),
        dtype=attention_mask.dtype,
        device=attention_mask.device,
    )

    # 合并历史 mask 和当前 mask
    attention_mask = torch.cat([past_mask, attention_mask], dim=-1)
```

### 5.3.3 past_key_values 参数

```python
outputs = self.model.generate(
    ...,
    past_key_values=past_key_values,  # 传入历史 Cache
    ...,
)

# 返回更新后的 Cache（可用于下一次生成）
new_past_key_values = outputs.past_key_values
```

## 5.5 答案提取函数

模型生成的文本可能包含多种内容，需要从中提取出**标准答案**。

### 5.5.1 extract_gsm8k_answer() —— 从 `\boxed{}` 提取

```python
def extract_gsm8k_answer(text: str) -> Optional[str]:
    # 匹配 \boxed{...} 格式
    boxes = re.findall(r"\\boxed\{([^}]*)\}", text)
    if boxes:
        content = boxes[-1]  # 取最后一个
        # 提取数字
        number = re.search(r"[-+]?\d+(?:\.\d+)?", content)
        return number.group(0) if number else content.strip()

    # Fallback: 提取最后一个数字
    numbers = re.findall(r"[-+]?\d+(?:\.\d+)?", text)
    if numbers:
        return numbers[-1]
    return None
```

**正则表达式解释**：

| 模式 | 含义 |
|------|------|
| `\\boxed\{` | 匹配 `\boxed{` |
| `([^}]*)` | 捕获花括号内的内容（非贪婪）|
| `[-+]?\d+` | 可选正负号 + 数字 |
| `(?:\.\d+)?` | 可选小数部分 |

**示例**：

```
输入: "解题过程... \boxed{42} \boxed{100}"
输出: "42"  (取最后一个)

输入: "答案是 3.14，圆周率约等于这个数"
输出: "3.14"  (取最后一个数字)
```

### 5.5.2 extract_gold() —— 从 GSM8K solution 提取

```python
def extract_gold(text: str) -> Optional[str]:
    # 匹配 "#### 答案" 格式
    match = re.search(r"####\s*([-+]?\d+(?:\.\d+)?)", text)
    return match.group(1) if match else None
```

**GSM8K 答案格式**：

```
解题过程...
5 + 3 = 8
#### 8
```

**正则表达式**：

| 模式 | 含义 |
|------|------|
| `####` | 匹配分隔符 |
| `\s*` | 0 或多个空白 |
| `([-+]?\d+(?:\.\d+)?)` | 捕获答案 |

### 5.5.3 normalize_answer() —— 答案标准化

```python
def normalize_answer(ans: Optional[str]) -> Optional[str]:
    if ans is None:
        return None
    return ans.strip().lower()
```

**标准化操作**：

| 操作 | 示例 |
|------|------|
| `.strip()` | `" 42 "` → `"42"` |
| `.lower()` | `"A"` → `"a"` |

**为什么需要标准化？**

模型可能输出：
- `Answer: 42`
- `The answer is 42`
- `42`

标准化后都变成 `42`，便于与标准答案匹配。

### 5.5.4 extract_markdown_python_block() —— 提取代码

```python
def extract_markdown_python_block(text: str) -> Optional[str]:
    # 匹配 ```python ... ``` 代码块
    pattern = r"```python(.*?)```"
    matches = re.findall(pattern, text, re.DOTALL | re.IGNORECASE)
    if matches:
        return matches[-1].strip()  # 取最后一个代码块
    return None
```

**正则表达式标志**：

| 标志 | 含义 |
|------|------|
| `re.DOTALL` | `.` 匹配换行符 |
| `re.IGNORECASE` | 忽略大小写 |

**示例**：

## 5.6 带超时的代码执行

### 5.6.1 为什么需要超时机制？

代码生成任务中，模型可能生成**无限循环**的代码。如果没有超时机制，程序会卡死。

### 5.6.2 run_with_timeout() 实现

```python
def run_with_timeout(code, timeout):
    def worker(ns, code):
        try:
            local_ns = {}
            exec(code, local_ns)
            ns['ok'] = True
            ns['error'] = None
        except Exception:
            ns['ok'] = False
            ns['error'] = traceback.format_exc()

    with Manager() as manager:
        ns = manager.dict()
        p = Process(target=worker, args=(ns, code))  # 子进程执行
        p.start()
        p.join(timeout)  # 等待超时

        if p.is_alive():  # 进程仍在运行
            p.terminate()  # 强制终止
            ns['ok'] = False
            ns['error'] = f"TimeoutError: Execution exceeded {timeout} seconds"

        return ns.get('ok', False), ns.get('error', None)
```

### 5.6.3 执行流程图

```
run_with_timeout() 执行流程：

1. 创建子进程
   │
   └── p = Process(target=worker, args=(ns, code))

2. 启动子进程
   │
   └── p.start()

3. 等待超时
   │
   └── p.join(timeout)
          │
          ├── 正常完成 ──→ 检查 ns['ok']
          │                    │
          │                    ├── True  → 返回 (True, None)
          │                    └── False → 返回 (False, error)
          │
          └── 超时 ──→ 终止进程
                        │
                        └── 返回 (False, "TimeoutError: ...")

4. 清理
   └── Manager 上下文退出
```

### 5.6.4 使用示例

```python
code = """
def solution():
    return sum(range(100))

result = solution()
"""

ok, error = run_with_timeout(code, timeout=5)

if ok:
    print("代码执行成功")
else:
    print(f"代码执行失败: {error}")
```

## 5.9 常见问题

### Q1：生成结果为空怎么办？

**原因**：
- `max_new_tokens` 太小
- 输入包含太多特殊 token
- 模型输出被 `skip_special_tokens=True` 跳过

**解决方案**：
```python
# 1. 增加 max_new_tokens
generate_text_batch(..., max_new_tokens=2048)

# 2. 检查原始输出
text = tokenizer.decode(generated_ids, skip_special_tokens=False)
print(text)
```

### Q2：答案提取失败怎么办？

**原因**：
- 模型输出格式不标准
- 缺少 `\boxed{}` 或 `####` 标记

**解决方案**：
```python
# 查看原始输出
print(model_output)

# 调整提取逻辑
answer = extract_gsm8k_answer(model_output) or model_output.split()[-1]
```

### Q3：代码执行超时怎么处理？

**原因**：
- 代码有无限循环
- 计算量太大

**解决方案**：
```python
ok, error = run_with_timeout(code, timeout=10)
if not ok:
    print(f"执行失败: {error}")
```
